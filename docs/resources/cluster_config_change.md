# typesense_cluster_config_change (Resource)

Schedules a configuration change for a Typesense Cloud cluster. Use this resource to modify cluster capacity (memory, vCPUs), high availability settings, or Typesense version.

Configuration changes can be scheduled for immediate execution or for a specific time in the future.

~> **Important:** This resource requires the `cloud_management_api_key` to be configured in the provider.

~> **Note:** Configuration changes cannot be updated after creation. To schedule a different change, delete and recreate the resource.

## Example Usage

### Upgrade Memory

```terraform
resource "typesense_cluster_config_change" "upgrade_memory" {
  cluster_id = typesense_cluster.production.id
  new_memory = "8_gb"
}
```

### Upgrade Typesense Version

```terraform
resource "typesense_cluster_config_change" "upgrade_version" {
  cluster_id                   = typesense_cluster.production.id
  new_typesense_server_version = "28.0"
}
```

### Enable High Availability

```terraform
resource "typesense_cluster_config_change" "enable_ha" {
  cluster_id           = typesense_cluster.production.id
  new_high_availability = "yes"
}
```

### Scheduled Upgrade

```terraform
resource "typesense_cluster_config_change" "scheduled_upgrade" {
  cluster_id        = typesense_cluster.production.id
  new_memory        = "16_gb"
  new_vcpu          = "8_vcpus"
  perform_change_at = 1704067200  # Unix timestamp for scheduled time
}
```

### Full Configuration Change

```terraform
resource "typesense_cluster_config_change" "full_upgrade" {
  cluster_id                   = typesense_cluster.production.id
  new_memory                   = "16_gb"
  new_vcpu                     = "8_vcpus"
  new_high_availability        = "yes_3_way"
  new_typesense_server_version = "28.0"
}
```

## Import

Cluster configuration changes can be imported using the change ID:

```shell
terraform import typesense_cluster_config_change.upgrade_memory change123
```
