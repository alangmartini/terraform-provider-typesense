# typesense_cluster (Resource)

Manages a Typesense Cloud cluster. This resource allows you to create, configure, and manage hosted Typesense clusters on Typesense Cloud.

~> **Important:** This resource requires the `cloud_management_api_key` to be configured in the provider.

~> **Note:** API keys (admin and search) are only available at cluster creation time. Ensure your Terraform state is properly secured.

## Example Usage

### Basic Cluster

```terraform
resource "typesense_cluster" "basic" {
  name                     = "my-cluster"
  memory                   = "1_gb"
  vcpu                     = "2_vcpus_4_hr_burst_per_day"
  typesense_server_version = "27.1"
  regions                  = ["us-east-1"]
}

output "cluster_endpoint" {
  value = typesense_cluster.basic.load_balanced_hostname
}
```

### Production Cluster with High Availability

```terraform
resource "typesense_cluster" "production" {
  name                     = "production"
  memory                   = "8_gb"
  vcpu                     = "4_vcpus"
  high_availability        = "yes"
  typesense_server_version = "27.1"
  regions                  = ["us-east-1", "us-west-2"]
  search_delivery_network  = "on"
  auto_upgrade_capacity    = true
}

output "production_endpoint" {
  value = typesense_cluster.production.load_balanced_hostname
}

output "admin_api_key" {
  value     = typesense_cluster.production.admin_api_key
  sensitive = true
}

output "search_api_key" {
  value     = typesense_cluster.production.search_api_key
  sensitive = true
}
```

### Development Cluster

```terraform
resource "typesense_cluster" "development" {
  name                     = "dev-cluster"
  memory                   = "0.5_gb"
  vcpu                     = "2_vcpus_4_hr_burst_per_day"
  high_availability        = "no"
  typesense_server_version = "27.1"
  regions                  = ["us-east-1"]
  search_delivery_network  = "off"
}
```

## vCPU Options

| Value | Description |
|-------|-------------|
| `2_vcpus_4_hr_burst_per_day` | 2 vCPUs with burst capacity |
| `2_vcpus` | 2 dedicated vCPUs |
| `4_vcpus` | 4 dedicated vCPUs |
| `8_vcpus` | 8 dedicated vCPUs |
| `16_vcpus` | 16 dedicated vCPUs |
| `32_vcpus` | 32 dedicated vCPUs |

## High Availability Options

| Value | Description |
|-------|-------------|
| `no` | Single node (no HA) |
| `yes` | 3-node cluster |
| `yes_3_way` | 3-node cluster (explicit) |
| `yes_5_way` | 5-node cluster |

## Import

Clusters can be imported using the cluster ID:

```shell
terraform import typesense_cluster.production abc123xyz
```

Note: When importing, API keys will not be available as they are only returned at creation time.
